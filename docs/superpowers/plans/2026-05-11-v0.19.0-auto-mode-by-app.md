# v0.19.0 — Per-mode auto-switching by app Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` (recommended) or `superpowers:executing-plans` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace v0.18.8's global `autoModeEnabled` toggle backed by static `apps.json` with **per-mode app lists** that users populate via a native macOS picker — making auto-mode customizable and matching the user's mental model ("which apps should this mode kick in for?").

**Architecture:** Each `VoiceMode` gains an optional `autoModeApps` array. At record-time, when the global `autoModeEnabled` is on, `selectModeForRecording` iterates `settings.modes` in order and returns the first mode whose `autoModeApps` includes the frontmost bundle ID. The existing curated `apps.json` becomes seed data only — copied into per-mode lists once on v0.19.0 first-launch (marker: `autoModeSeeded`). UI is a new `AutoModeAppsSection` inside the Mode editor; "+ Add app…" invokes a native `NSOpenPanel` filtered to `.app` bundles via a new Rust Tauri command.

**Tech Stack:** React 19 + TypeScript + Vite (frontend), Rust 2021 + Tauri 2 + `objc2-app-kit` / `objc2-foundation` (native picker), `tauri-plugin-store` (settings), Vitest (frontend tests), `cargo test` (Rust tests).

**Version target:** **v0.19.0** (minor bump — locked-in user decision per spec). Bump version once at the end of the plan, not on intermediate commits.

**Branching:** Work happens on `v0.19.0-auto-mode-by-app` branched off `main` (at the v0.18.12 ship point or later). Merge to `main` via fast-forward at the end.

**Out of scope (per spec):**
- Per-app custom mode params (sub-mode parameterization)
- Window-class / URL-pattern matching (browser tab → mode)
- iCloud sync of per-mode app lists across user's Macs

### Task 7: Rewrite `selectModeForRecording` to use per-mode `autoModeApps`

**Files:**

- [ ] **Step 1: Update the `selectModeForRecording` implementation**

Replace the existing function body (the v0.18.8 version that calls `pickAutoMode`) with the v0.19.0 per-mode-list lookup:

```ts
/**
 * Pick the mode for a single recording, gated on the user's opt-in setting.
 *
 * v0.19.0: when `autoModeEnabled` is true, iterate `modes` in order and
 * return the first mode whose `autoModeApps` includes the frontmost
 * bundle ID (case-insensitive). Mode order is user-controlled via
 * the drag handle in ModesPanel, so ties resolve deterministically.
 * Falls back to `activeModeId` when no mode matches or no bundle is
 * available. When `autoModeEnabled` is false, `activeModeId` always wins.
 *
 * Previously (v0.18.8) this called `pickAutoMode` which used the static
 * `apps.json` registry. v0.19.0 turns `apps.json` into seed-only data
 * (copied into per-mode `autoModeApps` lists once on first launch via
 * `migrateSeedAutoModeApps` in store-bridge.ts), so `pickAutoMode` is
 * no longer called at record time. It stays exported because the
 * seed migration imports it.
 */
export function selectModeForRecording(
  modes: VoiceMode[],
  activeModeId: string,
  frontmostBundleId: string | null | undefined,
  autoModeEnabled: boolean,
): VoiceMode {
  if (autoModeEnabled && frontmostBundleId) {
    const bid = frontmostBundleId.toLowerCase();
    const matched = modes.find((m) =>
      m.autoModeApps?.some((entry) => entry.bundleId.toLowerCase() === bid),
    );
    if (matched) return matched;
  }
  return modes.find((m) => m.id === activeModeId) ?? modes[0]!;
}
```

```ts
import appsJson from "./apps.json";

interface AppsRegistryEntry {
  bundleId: string;
  preferredMode: string;
  displayName: string;
}
interface AppsRegistry {
  version: number;
  apps: AppsRegistryEntry[];
}
const appsRegistry: AppsRegistry = appsJson as AppsRegistry;

/**
 * v0.18.x → 0.19.0 one-shot seed migration. Before v0.19.0 the
 * `apps.json` curated registry was the runtime source-of-truth for
 * auto-mode (bundle ID → preferredMode lookup). v0.19.0 makes
 * `apps.json` seed-only: walk it once, look up each `preferredMode` in
 * the user's saved modes, and append `{ bundleId, displayName }` to
 * that mode's `autoModeApps` (de-duped by bundle ID).
 *
 * Decision based on `settings.autoModeSeeded`. DEFAULT_SETTINGS now
 * seeds it true (fresh installs ship with DEFAULT_MODES already
 * including their curated `autoModeApps`), so the merged result always
 * carries the marker — checking the **stored** value directly is the
 * only way to detect a true v0.18.x-era settings file. Same pattern
 * as migratePillPositions.
 */
function migrateSeedAutoModeApps(
  storedRaw: Partial<AppSettings> | null | undefined,
  merged: AppSettings,
): { settings: AppSettings; changed: boolean } {
  if (!storedRaw) return { settings: merged, changed: false };
  if (storedRaw.autoModeSeeded) return { settings: merged, changed: false };
  // Pre-v0.19.0 save: walk apps.json and append to matching modes.
  const next = { ...merged };
  next.modes = merged.modes.map((m) => {
    const seedEntries = appsRegistry.apps.filter((a) => a.preferredMode === m.id);
    if (seedEntries.length === 0) {
      // Ensure the field exists even when no curated entries — empty
      // array beats undefined for downstream code clarity.
      return { ...m, autoModeApps: m.autoModeApps ?? [] };
    }
    const existing = m.autoModeApps ?? [];
    const existingBundleIds = new Set(existing.map((e) => e.bundleId.toLowerCase()));
    const toAdd = seedEntries
      .filter((s) => !existingBundleIds.has(s.bundleId.toLowerCase()))
      .map((s) => ({ bundleId: s.bundleId, displayName: s.displayName }));
    return { ...m, autoModeApps: [...existing, ...toAdd] };
  });
  next.autoModeSeeded = true;
  return { settings: next, changed: true };
}
```

```rust
#![cfg(target_os = "macos")]

//! Native `.app` bundle picker via NSOpenPanel. Used by the per-mode
//! auto-switching UI in v0.19.0: user clicks "+ Add app…" in the mode
//! editor, the modal panel opens scoped to `.app` bundles, user picks
//! one (or many), we return their bundle identifiers + display names
//! to JS for storage in the mode's `autoModeApps` list.
//!
//! Strategy B from docs/v0.19.0-auto-mode-by-app.md (NSOpenPanel filtered
//! to .app). Pros: native multi-select, native icons, no AppleScript
//! prompt. Cons: requires objc2-app-kit + objc2-foundation imports.

use objc2::rc::autoreleasepool;
use objc2::runtime::AnyObject;
use objc2::{class, msg_send};
use objc2_app_kit::NSOpenPanel;
use objc2_foundation::{NSArray, NSString, NSURL};
use serde::Serialize;

#[derive(Debug, Serialize)]
pub struct PickedApp {
    pub bundle_id: String,
    pub display_name: String,
    /// Absolute path to the picked .app bundle. The JS caller may
    /// resolve the icon separately (out of scope for v0.19.0).
    pub path: String,
}

#[tauri::command]
pub fn pick_app_bundle(prompt: String) -> Result<Vec<PickedApp>, String> {
    autoreleasepool(|_| unsafe {
        let panel: *mut NSOpenPanel = msg_send![class!(NSOpenPanel), openPanel];
        if panel.is_null() {
            return Err("NSOpenPanel returned null".to_string());
        }

        // Configure the panel
        let _: () = msg_send![panel, setAllowsMultipleSelection: true];
        let _: () = msg_send![panel, setCanChooseFiles: true];
        let _: () = msg_send![panel, setCanChooseDirectories: false];
        let _: () = msg_send![panel, setResolvesAliases: true];
        let title = NSString::from_str(&prompt);
        let _: () = msg_send![panel, setMessage: &*title];

        // Restrict to .app bundles
        let app_ext = NSString::from_str("app");
        let allowed: *mut AnyObject = msg_send![class!(NSArray), arrayWithObject: &*app_ext];
        let _: () = msg_send![panel, setAllowedFileTypes: allowed];

        // Default to /Applications
        let apps_path = NSString::from_str("/Applications");
        let apps_url: *mut AnyObject = msg_send![class!(NSURL), fileURLWithPath: &*apps_path];
        let _: () = msg_send![panel, setDirectoryURL: apps_url];

        // Run modally. NSModalResponseOK == 1.
        let response: isize = msg_send![panel, runModal];
        if response != 1 {
            return Ok(vec![]); // User cancelled — empty list is success.
        }

        let urls: *mut AnyObject = msg_send![panel, URLs];
        let count: usize = msg_send![urls, count];
        let mut out = Vec::with_capacity(count);
        for i in 0..count {
            let url: *mut NSURL = msg_send![urls, objectAtIndex: i];
            let bundle: *mut AnyObject = msg_send![class!(NSBundle), bundleWithURL: url];
            if bundle.is_null() {
                continue;
            }
            let bundle_id_ptr: *mut NSString = msg_send![bundle, bundleIdentifier];
            let display_name_key = NSString::from_str("CFBundleDisplayName");
            let display_name_ptr: *mut NSString =
                msg_send![bundle, objectForInfoDictionaryKey: &*display_name_key];
            // Fallback: if CFBundleDisplayName isn't set, use CFBundleName.
            let display_name_ptr: *mut NSString = if display_name_ptr.is_null() {
                let key = NSString::from_str("CFBundleName");
                msg_send![bundle, objectForInfoDictionaryKey: &*key]
            } else {
                display_name_ptr
            };
            let path_ptr: *mut NSString = msg_send![url, path];

            let bundle_id = if bundle_id_ptr.is_null() {
                continue; // No bundle ID → useless to us
            } else {
                (*bundle_id_ptr).to_string()
            };
            let display_name = if display_name_ptr.is_null() {
                bundle_id.clone() // Fallback to bundle ID as display name
            } else {
                (*display_name_ptr).to_string()
            };
            let path = if path_ptr.is_null() {
                String::new()
            } else {
                (*path_ptr).to_string()
            };
            out.push(PickedApp { bundle_id, display_name, path });
        }
        Ok(out)
    })
}
```

```tsx
import { tokens, Row, Section, Button } from "./ui";
import { useT } from "../lib/i18n/I18nProvider";
import { pickAppBundle } from "../lib/tauri-bridge";
import type { VoiceMode } from "../lib/voiceModes";

interface AutoModeAppsSectionProps {
  mode: VoiceMode;
  onChange: (next: NonNullable<VoiceMode["autoModeApps"]>) => void;
}

/**
 * UI section inside the Mode editor for managing per-mode auto-switch
 * apps. List of pills with × remove buttons + an "Add app…" button
 * that triggers a native NSOpenPanel via the pick_app_bundle Tauri
 * command. v0.19.0.
 */
export function AutoModeAppsSection({ mode, onChange }: AutoModeAppsSectionProps) {
  const t = useT();
  const apps = mode.autoModeApps ?? [];

  const handleAdd = async () => {
    const prompt = t.panels.modes.autoModeAppsPickerTitle(mode.name);
    const picked = await pickAppBundle(prompt);
    if (picked.length === 0) return; // user cancelled
    // De-dupe by bundle ID (case-insensitive)
    const existingIds = new Set(apps.map((a) => a.bundleId.toLowerCase()));
    const newApps = picked
      .filter((p) => !existingIds.has(p.bundleId.toLowerCase()))
      .map((p) => ({ bundleId: p.bundleId, displayName: p.displayName }));
    if (newApps.length === 0) return; // all picks were duplicates
    onChange([...apps, ...newApps]);
  };

  const handleRemove = (bundleId: string) => {
    onChange(apps.filter((a) => a.bundleId !== bundleId));
  };

  return (
    <Section
      label={t.panels.modes.autoModeAppsLabel}
      help={t.panels.modes.autoModeAppsHelp(mode.name)}
    >
      <div className="flex flex-col gap-1.5">
        {apps.length === 0 && (
          <p
            className="text-[12px] italic px-1 py-2"
            style={{ color: tokens.fgSubtle }}
          >
            {t.panels.modes.autoModeAppsEmpty}
          </p>
        )}
        {apps.map((app) => (
          <div
            key={app.bundleId}
            className="flex items-center justify-between rounded-md px-2.5 py-1.5"
            style={{
              background: tokens.control,
              border: `1px solid ${tokens.border}`,
            }}
          >
            <span style={{ fontSize: 12, color: tokens.fg }}>{app.displayName}</span>
            <button
              type="button"
              aria-label={t.panels.modes.autoModeAppsRemove(app.displayName)}
              onClick={() => handleRemove(app.bundleId)}
              style={{
                background: "none",
                border: "none",
                color: tokens.fgMuted,
                cursor: "pointer",
                fontSize: 14,
                padding: "0 4px",
              }}
              onMouseEnter={(e) => { e.currentTarget.style.color = "var(--color-warning)"; }}
              onMouseLeave={(e) => { e.currentTarget.style.color = tokens.fgMuted; }}
            >
              ✕
            </button>
          </div>
        ))}
        <div className="pt-1">
          <Button size="xs" variant="outline" onClick={() => void handleAdd()}>
            {t.panels.modes.autoModeAppsAdd}
          </Button>
        </div>
      </div>
    </Section>
  );
}
```
