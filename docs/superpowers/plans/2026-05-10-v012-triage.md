## Phase A — Correctness bugs (priority order)

### Task A1: Fix cleanup preamble that fights markdown formatting

**Symptom (from user testing):** Note mode with explicit "create a list / headline" instruction produced plain prose under both **Local on-device LLM (Plus)** and **Claude Code CLI (Sonnet Balanced)** providers. Cleanup pipeline IS being called — the prompt is getting through — but the output strips markdown.

**Root cause hypothesis:** The hard-coded preamble at [transcribe.ts:185-191](apps/ultravox/src/lib/transcribe.ts:185) says:

> *"Output ONLY the cleaned text. No preamble, **no Markdown fences**, no quotes."*

A safety-conscious LLM (especially Sonnet) reads "no Markdown fences" as "no markdown at all" — particularly when combined with *"NEVER paraphrase or rewrite in your own words. Preserve the speaker's voice."* The two rules conjoin into "output verbatim prose, no formatting." Per-style templates that request `# headings` and `- bullets` lose against the preamble.

**Fix direction:** Tighten the preamble to forbid only code fences (which IS the safety concern — preventing markdown injection or wrapping), and explicitly permit structural markdown when the per-style template asks for it.

**Files:**
- Modify: [apps/ultravox/src/lib/transcribe.ts:185-191](apps/ultravox/src/lib/transcribe.ts:185)
- Test: `apps/ultravox/src/tests/buildClaudePrompt.test.ts` (new)

### Task A3: Disable echo cancellation to stop ducking background music

**Symptom:** When recording starts, background music (Spotify / Apple Music) ducks — even when the user has *not* enabled `pauseMediaWhileRecording`. macOS WebKit's `getUserMedia` defaults `echoCancellation: true`, which routes the mic through the Voice-Processing IO Audio Unit. That unit raises the audio session's "voice processing" priority and the system ducks all other audio.

**Decision (per user):** Remove echo cancellation entirely. We are NOT a video-call app — there's no playback returning to the mic to cancel. Document the decision in CLAUDE.md.

#[derive(serde::Serialize)]
#[serde(rename_all = "lowercase")]
pub enum MicAuth {
    NotDetermined,
    Restricted,
    Denied,
    Authorized,
}

#[tauri::command]
pub fn microphone_auth_status() -> MicAuth {
    let status = unsafe {
        AVCaptureDevice::authorizationStatusForMediaType(AVMediaTypeAudio)
    };
    match status {
        AVAuthorizationStatus::NotDetermined => MicAuth::NotDetermined,
        AVAuthorizationStatus::Restricted    => MicAuth::Restricted,
        AVAuthorizationStatus::Denied        => MicAuth::Denied,
        AVAuthorizationStatus::Authorized    => MicAuth::Authorized,
        _ => MicAuth::NotDetermined,
    }
}

### Task A5: Paste reliability — fix the "V" bug

- [ ] **Step 2: Implement the atomic dispatcher**

```rust
fn dispatch_paste_combo_atomic() -> Result<(), String> {
    use core_graphics::event::{CGEvent, CGEventFlags, CGEventTapLocation};
    use core_graphics::event_source::{CGEventSource, CGEventSourceStateID};

    const KEY_V: u16 = 9; // ANSI 'V' keycode
    let source = CGEventSource::new(CGEventSourceStateID::CombinedSessionState)
        .map_err(|_| "CGEventSource".to_string())?;

    let down = CGEvent::new_keyboard_event(source.clone(), KEY_V, true)
        .map_err(|_| "keyDown".to_string())?;
    down.set_flags(CGEventFlags::CGEventFlagMaskCommand);
    down.post(CGEventTapLocation::HID);

    let up = CGEvent::new_keyboard_event(source, KEY_V, false)
        .map_err(|_| "keyUp".to_string())?;
    up.set_flags(CGEventFlags::CGEventFlagMaskCommand);
    up.post(CGEventTapLocation::HID);

    Ok(())
}
```

pub fn activate_app_by_pid(pid: i32) -> Result<(), String> {
